# Day 24 – Advanced Git: Merge, Rebase, Stash & Cherry Pick

## Overview

Today I learned advanced Git concepts that are essential for real-world development and DevOps workflows.

## Topics Practiced

### Git Merge

* Performed Fast-Forward Merge
* Performed Merge Commit
* Created and Resolved Merge Conflicts

### Git Rebase

* Rebased a feature branch onto main
* Observed linear commit history
* Compared Rebase vs Merge

### Squash Merge

* Combined multiple commits into a single commit
* Compared Squash Merge with Regular Merge

### Git Stash

* Saved uncommitted work using stash
* Restored changes using stash pop
* Worked with multiple stashes

### Cherry Pick

* Applied a specific commit from one branch to another
* Resolved cherry-pick conflicts
* Completed cherry-pick using `git cherry-pick --continue`

## Commands Practiced

```bash
git merge
git rebase
git merge --squash
git stash
git stash list
git stash pop
git cherry-pick
git cherry-pick --continue
git log --oneline --graph --all
```

## Key Learnings

* Fast-forward merge keeps history simple.
* Merge commits preserve branch history.
* Rebase creates a clean linear history.
* Squash merge combines multiple commits into one.
* Stash helps save work temporarily.
* Cherry-pick applies specific commits without merging the entire branch.
* Merge and cherry-pick conflicts can be resolved manually.
